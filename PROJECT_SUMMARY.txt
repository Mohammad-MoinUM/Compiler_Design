================================================================================
UNIQUE FEATURE: OPERATOR REMAPPING
================================================================================

BLPL implements semantic remapping of operators to demonstrate design principle:
"Operators are syntax; semantics are defined separately"

MAPPING TABLE:
┌──────────────┬─────────────────┬──────────────────┐
│ BLPL Syntax  │ BLPL Semantics  │ Generated C Code │
├──────────────┼─────────────────┼──────────────────┤
│ x + y        │ Multiply        │ x * y            │
│ x - y        │ Divide          │ x / y            │
│ x * y        │ Add             │ x + y            │
│ x / y        │ Subtract        │ x - y            │
└──────────────┴─────────────────┴──────────────────┘

EXAMPLE VERIFICATION:
  Input BLPL:  dhoro result = 5 + 3;  // Says "add"
  Actual Op:   5 * 3 = 15              // But does multiply!
  Generated C: int result = (5 * 3);
  Output:      result = 15 ✓

COMPILATION INSTRUCTIONS FOR TEACHER

TO REBUILD FROM SCRATCH:

1. Clean previous builds:
   cd e:\Semesters\3-2\3-2 projects\compiler
   del *.o lex.yy.c bangla_lover.tab.c bangla_lover.tab.h

2. Generate parser and lexer:
   bison -d bangla_lover.y
   flex bangla_lover.l

3. Compile:
   gcc -c -Wall -g bangla_lover.tab.c
   gcc -c -Wall -g lex.yy.c
   gcc -c -Wall -g symbol_table.c

4. Link:
   gcc -o blpl_compiler.exe bangla_lover.tab.o lex.yy.o symbol_table.o -lm

5. Test:
   blpl_compiler.exe test_simple.bl output.txt
   gcc -o test.exe blpl_output.c -lm
   ./test.exe
